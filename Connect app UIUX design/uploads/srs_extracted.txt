1.  Introduction
1.1  Purpose
This Software Requirements Specification (SRS) defines the complete functional and non-functional requirements for Recall — a cross-platform networking application comprising a mobile app (iOS & Android) and a browser extension (Chrome & Safari). Recall lets professionals create a rich Digital Business Card (DBC), exchange it instantly via NFC tap, QR code, Airdrop, or link, and automatically logs every contact exchange with contextual metadata — when, where, and how they met. An AI-powered chat assistant lets users query their contact history in plain language.

1.3  Scope
Recall consists of three deliverables: (1) Recall Mobile App — primary networking tool for iOS and Android; (2) Recall Browser Extension — desktop companion for Chrome and Safari; (3) Recall Cloud Backend — REST API, real-time sync, AI search, and analytics.

Out of scope for v1.0: physical NFC card manufacturing, payment processing, CRM integrations (v2.0), video calling.

2.  Overall Description
2.1  Product Perspective
Recall is a standalone SaaS product replacing physical business cards and passive contact apps. It combines identity sharing, contextual logging, and AI-powered recall into a single platform. The name 'Recall' reflects the core promise: every person you meet is instantly findable — by name, face, location, role, or even a vague memory.

2.3  Design Constraints
NFC tap requires NFC-capable hardware — graceful fallback to QR on unsupported devices.
Airdrop is Apple-only; Nearby Share is Android-only; QR/link is universal cross-platform.
Browser extension cannot access device NFC; uses QR and link sharing only.
All personal data must comply with GDPR and PDPA Pakistan 2023.
Offline mode required — full contact access and DBC display with no internet connection.

3.5  AI Contact Intelligence (Chat Search)
Users open a chat interface and describe who they are looking for in plain language. The AI searches all contact metadata and returns ranked, contextually explained results.

3.5.1  Sample Interaction

You   Who did I meet at the startup expo in Karachi last month?

Recall AI   I found 4 contacts from an event near Expo Centre Karachi on April 8–9, 2026: (1) Sara Malik, Head of Product @ Airlift — met via NFC, note: 'discuss API partnership'; (2) Usman Qureshi, CTO @ Bykea — QR scan; (3) Farhan Ahmed, VC @ Sarmayacar — tagged Investor; (4) Nadia Hussain, UX Lead @ Systems Ltd. Want to filter by role?

You   Show me only the investors

Recall AI   1 investor from that event: Farhan Ahmed, VC @ Sarmayacar — met April 9, Karachi. Tagged Investor, no follow-up note recorded. Want me to draft a follow-up message?

4.2  Security
All data in transit: TLS 1.3; HSTS enforced on all web endpoints.
All data at rest: AES-256 (cloud), device keychain/keystore (local).
API rate limiting: 100 requests/min per authenticated user.
OWASP Top 10 addressed and verified via penetration testing before launch.
Contact data never shared with third parties or used for advertising.
4.3  Privacy & Compliance
GDPR-compliant: consent management, right to erasure, data portability (Article 20).
PDPA Pakistan 2023 compliance for Pakistani users.
Location data: explicit opt-in per session; retroactively disablable.
AI embeddings stored only in the user's encrypted partition; raw PII never sent externally.
4.4  Reliability
Backend API target uptime: 99.9% (< 8.7 hours downtime/year).
Full offline mode: contacts and DBC cached locally.
Daily automated backups; 30-day point-in-time recovery.
AI feature degrades gracefully to keyword search if AI service is unavailable.

5.2  AI Architecture — Recall Search
Step 1 — Index: On every contact add/edit, the Index Sync Worker generates an embedding from a sanitised contact text string and stores it in pgvector.
Step 2 — Query: User submits natural language query via chat UI.
Step 3 — Parse: LLM extracts structured filters {name, company, location, dateRange, tags, industry} from the raw query.
Step 4 — Search: Hybrid search — pgvector ANN (semantic) + PostgreSQL FTS (keyword) run in parallel, results merged.
Step 5 — Re-rank: Top 20 candidates scored by a cross-encoder against the original query.
Step 6 — Generate: Top results + query passed to response LLM → conversational reply with inline contact cards.

Embedding schema: '[Name] [Title] at [Company]. Met via [Method] on [Date] at [Location]. Tags: [Tags]. Note: [Note].' — this is the only text sent to external embedding APIs.
